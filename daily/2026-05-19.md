## AI 基础 — 提示词（Prompt）（70 分钟）

### 本章核心一句话

> Prompt 是**软约束**，不是安全边界。真正的边界必须由代码、权限、校验和审计来承担。

### 学习重点

**1. Prompt 到底是什么？（10 分钟）**
- 不只是「怎么问 AI」，而是把任务目标、输入边界、输出格式、失败处理和安全规则写进一次可执行的沟通协议
- 好的 Prompt 不是让模型更自信，而是让模型在合适的时候**停下来**

**2. 四个关键概念（20 分钟）**

| 概念 | 一句话理解 |
|------|-----------|
| Instruction | 给模型的任务规则：角色是什么、要做什么、不能做什么、输出什么格式 |
| Few-shot | 在 Prompt 里放少量示例，让模型模仿判断方式和输出格式 |
| Structured Output | 让模型按固定结构（如 JSON）返回结果，方便后续代码处理 |
| Prompt Injection | 攻击者通过输入内容让模型忽略原始规则，在 Agent 场景里尤其危险 |

**3. 写 Instruction 的四段结构（20 分钟）**
一个好的 Instruction 包含：
1. 任务目标
2. 可用输入
3. 禁止行为
4. 输出格式和失败格式

**4. 在 AI × Web3 里的位置（20 分钟）**
完整安全链路：
```
Prompt（定义任务）→ Context（可信数据）→ Model（生成结果）
→ Code（校验 schema）→ Guard/Simulation（检查链上影响）→ Human Check（确认高风险）
```
高风险动作（支付、签名、写入数据库）**不能只靠 Prompt 拦截**。

## Web3 基础 — 钱包（Wallet）（20 分钟）

### 本章核心一句话

> 钱包管理的不是「账号资料」，而是**链上控制权**。连接钱包 ≠ 授权资产。

### 学习重点（轻量阅读，建立直觉）

**三类操作，风险完全不同：**

| 操作 | 实际含义 | 风险级别 |
|------|----------|---------|
| 连接钱包 | 应用读取地址和网络，不能动用资产 | 低 |
| 签名消息 | 证明你控制某地址，可用于登录/授权/创建订单 | 中 |
| 发送交易 | 请求改变链上状态，可能转账、授权 token、调用合约 | 高 |

**五个基础概念：**

| 概念 | 一句话理解 |
|------|-----------|
| EOA | 由私钥控制的普通账户，是大多数钱包的默认类型 |
| 助记词（Mnemonic） | 恢复钱包的高风险秘密，任何要求你输入助记词的网页/客服都应视为危险 |
| Transaction | 对链上状态的正式请求，成功上链后通常不能撤回 |
| Gas | 链上执行成本，不足或估算不准会导致交易失败 |
| Explorer | 查看链上事实的工具（如 Etherscan），可验证交易是否真实成功 |
