## 学习节奏

- 每日投入：1–2 小时
- 核心方向：Hackathon 项目 + 产品/研究
- 工具策略：无代码背景，优先用 AI 工具（Claude/ChatGPT）辅助理解和操作

## 推荐学习路径

Handbook 建议：「正在准备 Hackathon，可以从**前沿探索**开始，把知识节点组合成小原型，而不是停留在概念讨论。」

根据你的背景（AI/Web3 有基础），建议以下顺序：

### 阶段一：补齐关键概念缺口（1–2 周）

重点读懂下列章节，建立跨界共同语言：

**AI 基础（你已有基础，重点看这几个）**
- [ ] [智能体（Agent）](https://aiweb3.school/zh/handbook/) — Agent 如何进入工具调用和多步执行
- [ ] [模型上下文协议（MCP）](https://aiweb3.school/zh/handbook/) — 模型、工具和上下文如何通过协议连接
- [ ] [氛围编程（Vibe Coding）](https://aiweb3.school/zh/handbook/) — 无代码背景必读，AI 辅助开发的方法论

**Web3 基础（你已有基础，重点看这几个）**
- [ ] [账户抽象（Account Abstraction）](https://aiweb3.school/zh/handbook/) — 为什么 Smart Account 更适合表达 Agent 权限
- [ ] [安全（Security）](https://aiweb3.school/zh/handbook/) — 合约、权限和风险边界

### 阶段二：进入 AI × Web3 Bridge（2–3 周）

这是真正的交叉问题所在：

- [ ] [链感知上下文（Chain-aware Context）](https://aiweb3.school/zh/handbook/) — 链上状态如何进入 Agent
- [ ] [Web3 工具调用（Web3 Tool Use）](https://aiweb3.school/zh/handbook/) — Agent 如何调用 RPC、钱包、合约工具
- [ ] [智能体工作流（Agent Workflow）](https://aiweb3.school/zh/handbook/) — 哪些步骤可以自动化，哪些需要人工确认
- [ ] [智能体钱包（Agent Wallet）](https://aiweb3.school/zh/handbook/) — Agent 能拿什么权限，如何限制和撤销
- [ ] [机器支付（Machine Payment）](https://aiweb3.school/zh/handbook/) — 机器之间如何完成小额支付和服务结算
- [ ] [AI 安全（AI Security）](https://aiweb3.school/zh/handbook/) — Prompt Injection、工具滥用和权限隔离

### 阶段三：前沿探索 + Hackathon 准备

选择最感兴趣的赛道深入：

- [ ] [智能体商业（Agentic Commerce）](https://aiweb3.school/zh/handbook/) — Agent 发现服务、协商任务、完成支付
- [ ] [钱包与权限（Wallet / Permission）](https://aiweb3.school/zh/handbook/) — Session Key、Policy 和 Guard 产品原型
- [ ] [治理（Governance）](https://aiweb3.school/zh/handbook/) — DAO、公共物品和协议治理的 AI 协作工具
- [ ] [开放赛道（Open Track）](https://aiweb3.school/zh/handbook/) — 新的交叉问题和还没被固定分类的方向

## 三条每日路径

### 最小路径（30–45 分钟）
1. 阅读 Handbook 一个章节（15 分钟）
2. 用自己的话写 1 条核心概念摘要
3. 更新 `daily/YYYY-MM-DD.md`

### 推荐路径（1–1.5 小时）
1. 读 Handbook 章节 + 相关资料（20 分钟）
2. 完成 WCB 平台当日任务（30 分钟）
3. 记录笔记和 1 个问题或卡点
4. 更新 daily note，生成打卡草稿

### 挑战路径（1.5–2 小时）
1. 推荐路径全部完成
2. 在 `experiments/` 或 `hackathon/` 下动手探索一个想法
3. 写一条 Handbook feedback（如有卡点或建议）
